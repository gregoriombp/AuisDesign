# Design System Setup — Claude Design Handoff

This skill takes a **Claude Design** handoff (a downloaded `.zip` **or** a fetch
URL passed to Claude Code) and sets up a **Next.js + shadcn/ui** project with
an `/auis/styleguide` route that documents tokens and components.

> **Fundamental rule:** always extract from the bundle. Do not regenerate anything via
> `shadcn add` — if the component does not exist in the handoff, record it as
> pending and ask the user before creating it from scratch.

---

## Input

Accepts one of two forms:

1. **A local `.zip` path** — the user pastes something like
   `~/Downloads/handoff-2026-04-28.zip` or drags the file in.
2. **Fetch URL** — a handoff link generated by Claude Design (e.g.
   `https://claude.com/design/handoff/...`). In this case, download it first:

   ```bash
   curl -L -o /tmp/claude-handoff.zip "<fetch_url>"
   ```

If the user provides neither, **ask** before continuing.

---

## Workflow

### 0. Inspect the bundle (DO NOT SKIP)

The internal structure of the Claude Design handoff is **neither public nor fixed**.
Do not assume file names. Always discover them dynamically:

```bash
mkdir -p /tmp/claude-handoff && cd /tmp/claude-handoff
unzip -o "<zip_path>"
unzip -l "<zip_path>"   # list the contents
find . -maxdepth 3 -type f | sort
```

Then classify what you found. Look for:

- **Manifest / spec** — any `*.json`, `manifest.*`, `handoff.*`,
  `design.*`, `bundle.*`, `tokens.*`, `CLAUDE.md`.
- **Tokens** — CSS with `--var:` (usually a `globals.css`,
  `tokens.css`, or inline in `*.html`), JSON with keys like `colors`,
  `typography`, `spacing`, `radius`, `shadow`.
- **Components** — `.tsx`/`.jsx`/`.html` files; `components/`,
  `ui/`, `src/` folders.
- **Pages** — `design.html` or several `.html` files representing states.
- **Assets** — a folder with `*.png`, `*.svg`, `*.webp`.
- **Notes** — `design-notes.md`, `README.md`, `notes/*.md`.

Before proceeding, **list in text** for the user:

- Total files found
- Where the tokens were detected (path + format: CSS / JSON / inline HTML)
- How many components were identified and in which language
  (`.tsx`, `.html`, etc.)
- Any `*.md` notes files

Only move on to step 1 after that quick confirmation.

---

### 1. Analyze the Design System

From what was found, identify and/or infer:

**Colors:**

- Primary / brand color → full scale (50–900+).
- Neutral/grey colors → full scale (50–900+).
- Semantic colors (success, error, warning, info).
- Background and surface.
- Border colors.
- Any other color present in the handoff.

**Typography:**

- Font family (sans-serif, serif, monospace, etc.).
- Heading sizes and weights.
- Body / label / caption sizes.
- Additional details (line-height, letter-spacing, font-feature-settings).

**Spacing & Radius:**

- Spacing rhythm (tight, normal, relaxed).
- Border radius style (sharp, rounded, pill).

**Shadows:**

- Shadow style (none, subtle, prominent).

**Important:** analyze ANY other aspect present in the bundle (motion,
z-index, breakpoints, opacities, focus rings, etc.). Do not limit the analysis
to what is above.

### 4. Install the recommended font

If the handoff indicates a font (in CSS, README or design-notes), install it via
`next/font`. Example:

```tsx
// /app/layout.tsx
import { Inter } from "next/font/google"; // replace with the handoff's font

const font = Inter({ subsets: ["latin"] });

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body className={font.className}>{children}</body>
    </html>
  );
}
```

---

### 5. Extract the components from the bundle

**ALWAYS extract from the `.zip`. Do not use `shadcn add` as a fallback.**

For each component identified in step 0:

1. Copy the file to `/components/ui/<name>.tsx` (or `/components/<name>.tsx`
   if it is composite).
2. Adapt the imports to the project alias (`@/components/ui/...`,
   `@/lib/utils`).
3. Replace hard-coded values with tokens (`var(--primary)`,
   `text-foreground`, etc.) whenever equivalent tokens exist.
4. If the component comes as **HTML** (common in Claude Design bundles),
   convert it to JSX/TSX preserving structure, classes, ARIA and states.
5. If the component uses external libs (Radix, lucide-react, etc.), install
   only the necessary deps — no `shadcn add`.

**If an expected component is missing** (e.g. the styleguide asks for `Button` but the
zip does not have it), stop and ask the user whether they want to:

- (a) extract it from another file in the bundle,
- (b) recreate it manually based on another component in the handoff, or
- (c) make an exception and use `shadcn add` for that specific component.

**Never** silently fall back to shadcn.

### 7. Create the styleguide layout with a sidebar

`/app/auis/styleguide/layout.tsx`:

```tsx
"use client";

import Link from "next/link";
import { usePathname } from "next/navigation";
import { cn } from "@/lib/utils";
import { navigation } from "./navigation";

export default function StyleguideLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  const pathname = usePathname();

  return (
    <div className="flex min-h-screen">
      <aside className="w-64 border-r bg-card p-6 flex flex-col gap-6 fixed top-0 left-0 h-screen overflow-y-auto">
        <div>
          <Link href="/auis/styleguide" className="text-xl font-bold">
            Design System
          </Link>
        </div>

        <nav className="flex flex-col gap-6">
          {navigation.map((section) => (
            <div key={section.title}>
              <h3 className="text-sm font-semibold text-muted-foreground mb-2">
                {section.title}
              </h3>
              <ul className="flex flex-col gap-1">
                {section.items.map((item) => (
                  <li key={item.href}>
                    <Link
                      href={item.href}
                      className={cn(
                        "block px-3 py-2 rounded-md text-sm transition-colors",
                        pathname === item.href
                          ? "bg-primary text-primary-foreground"
                          : "hover:bg-muted",
                      )}>
                      {item.name}
                    </Link>
                  </li>
                ))}
              </ul>
            </div>
          ))}
        </nav>
      </aside>

      <main className="flex-1 ml-64 overflow-auto">{children}</main>
    </div>
  );
}
```

### 8. Create the main styleguide page

`/app/auis/styleguide/page.tsx` — displays **every extracted token** on a single
page:

- **Color palette** — every color as a swatch, with the CSS variable name
  and the real value.
- **Primary scale** — 50 to 900 (or the handoff's equivalent).
- **Grey scale** — 50 to 900.
- **Semantic colors** — success, warning, error, info.
- **Typography** — heading and body samples with the extracted font.
- **Border radius** — examples of each size.
- **Shadows** — visual examples.
- **Components** — a preview of each extracted component.
- **Dark mode toggle** — a preview of both themes (only if the handoff carries
  dark).

> Include any extra token the handoff carries that is not in the list
> above (motion, z-index, breakpoints, opacities, etc.).

For each component, also create
`/app/auis/styleguide/components/<name>/page.tsx` with variants, states and
usage examples based on what came in the handoff.

## Notes

- If the bundle structure is ambiguous (several candidates for "canonical
  tokens", multiple `.css` files), **ask** before choosing.
- **Never** fill in missing tokens by inventing values. Mark them as a gap.
- **Never** install a component via `shadcn add` without explicit confirmation from the
  user, even if the standard styleguide asks for it.
- If the handoff carries a partial dark mode (only some variables), expose the
  toggle only if it covers enough not to break; otherwise,
  disable the toggle and mention it in the summary.
